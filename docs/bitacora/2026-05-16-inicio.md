# Bitácora — 2026-05-16 — Inicio del proyecto AQCS

## Contexto

Inicio formal del **Autonomous Quant Crypto System (AQCS)**.

El objetivo de esta sesión fue establecer la capa de fundación (Fase 1) del laboratorio cuantitativo. No se ha escrito ninguna lógica de trading. No se ha ejecutado ninguna orden. El trabajo de hoy es infraestructura pura.

## Lo que se hizo

### Estructura del repositorio

Se creó la estructura completa de directorios siguiendo una separación estricta de responsabilidades:

- `src/data/` — adquisición de datos de mercado
- `src/features/` — ingeniería de características (vacío, Fase 2)
- `src/signals/` — generación de señales (vacío, Fase 2)
- `src/portfolio/`, `src/risk/`, `src/execution/` — (vacíos, fases futuras)
- `src/backtesting/` — motor de simulación (vacío, Fase 2)
- `src/monitoring/` — salud del sistema (vacío, Fase 2)
- `src/utils/` — configuración, logging, esquema de eventos
- `src/llm_oversight/` — capa de observación LLM (solo lectura)

### Módulos implementados

**`src/utils/config.py`**  
Carga configuración desde YAML con merge jerárquico (base + override por entorno). Variables de entorno tipadas via Pydantic-Settings.

**`src/utils/logging.py`**  
Logging estructurado JSON usando structlog. Sin `print()` en ningún lugar del codebase.

**`src/utils/events.py`**  
Esquema de eventos tipados e inmutables con Pydantic. Categorías: DATA, FEATURE, SIGNAL, PORTFOLIO, RISK, SYSTEM, OVERSIGHT.

**`src/data/ohlcv.py`**  
Descargador de velas OHLCV desde Binance Spot vía ccxt. Pagina automáticamente, elimina duplicados, guarda en Parquet con schema PyArrow declarado.

**`src/llm_oversight/observer.py`**  
Recibe OversightEvents y escribe logs. No modifica estado. No toma decisiones. No llama APIs externas.

## Decisiones tomadas

1. **No asyncio en Fase 1** — la simplicidad síncrona es suficiente para descarga batch.
2. **Parquet sobre CSV** — schema enforcement, compresión, interoperabilidad con herramientas analíticas futuras.
3. **structlog sobre logging stdlib** — JSON nativo, contextvars, renderers intercambiables.
4. **uv como gestor de paquetes** — velocidad, compatibilidad con pyproject.toml estándar.
5. **Pydantic v2 para eventos** — inmutabilidad (`frozen=True`), validación Rust-backed.

## Invariantes de seguridad confirmados

- `features.order_execution: false` en configuración base.
- `features.autonomous_trading: false` en configuración base.
- `features.live_data: false` en configuración base.
- El módulo `llm_oversight` no tiene ningún pathway de escritura a exchange.
- No hay secretos hardcodeados en ningún archivo.
