## CLI Behavior

```bash
PYTHONPATH=src python scripts/data/run_public_ohlcv_burn_in.py \
  --exchange binance \
  --symbols BTCUSDT,ETHUSDT,SOLUSDT \
  --timeframe 1h \
  --limit 200 \
  --output-dir data/burn_in/
```

**Options:**

| Option | Values | Default |
|---|---|---|
| `--symbols` | Comma-separated: BTCUSDT, ETHUSDT, SOLUSDT | required |
| `--timeframe` | 1h | required |
| `--limit` | 1–500 | 200 |
| `--output-dir` | any path | required |
| `--exchange` | binance | binance |

**Exit codes:**

| Code | Meaning |
|---|---|
| 0 | All symbols passed |
| 1 | At least one symbol failed |
| 2 | Invalid arguments or config error |

**Stdout:** Deterministic JSON summary (all log output goes to stderr).

## Burn-In Output Structure

Per run, the output directory contains:

```text

  BTC_USDT_1h.parquet          — raw OHLCV data
```

## Per-Symbol Pipeline

For each symbol, `run_symbol_capture` runs:
1. **Fetch** — `exchange.fetch_ohlcv(symbol, timeframe, limit=N)` via public API
2. **Dedup** — `drop_duplicates(subset="timestamp_ms")` on raw candles
3. **Build DataFrame** — UTC-aware timestamps, OHLCV schema
4. **Validate** — `validate_ohlcv(df, symbol, timeframe)` — 13 structural checks
5. **Save Parquet** — `save_parquet(df, output_dir, symbol, timeframe)`
6. **Data quality** — `check_ohlcv_parquet_quality(parquet_path, timeframe)`
7. **Manifest** — `generate_manifest(parquet_path, symbol, timeframe)` + `save_manifest`
8. **Verify** — `verify_manifest(parquet_path, manifest)`

Failure in any step is contained to that symbol; remaining symbols continue.

## Test Strategy

All 43 tests use mocked API responses. No live network calls.

**Mocking approach:**

- Core functions (`run_burn_in`, `run_symbol_capture`): inject a `MagicMock` exchange
  via the `exchange` kwarg. The mock's `fetch_ohlcv.side_effect` dispatches per ccxt symbol.
- CLI (`main()`): patch `_build_public_exchange` to return the mock exchange.
- Validation failure tests: patch `_fetch_ohlcv` directly to inject DataFrames
  with duplicate or non-monotonic timestamps.

## Risks

- Same logging redirection pattern as smoke test: `main()` reconfigures structlog to stderr.
  Appropriate for a CLI entrypoint; does not affect library imports.
- Raw duplicate timestamps from the exchange are silently deduped by `_fetch_ohlcv` before
  `validate_ohlcv` runs. Consistent with existing `aqcs.data.ohlcv.fetch_ohlcv` behavior.
- Single symbol failure continues other symbols. If Binance is rate-limiting, partial failures
  may occur. The JSON summary clearly identifies `symbols_passed` vs `symbols_failed`.
